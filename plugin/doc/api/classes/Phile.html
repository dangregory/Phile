Phile Class

Defined in: src/Phile.js:148

The Phile class provides a javascript API to work with Pentaho files. The optional options parameter sets standard options for this Phile instance. Any global options present in Phile.options are applied if not present in options. Please checkout the defaultOptions property to see the supported options.

getChildren

(
  • conf
)

Defined in src/Phile.js:389

This method can be used to get the contents of a directory. The method is passed a configuration object to specify callbacks and the path from where to get children from. The configuration object supports all generic configuration properties as documented in the request() method. Specific properties:

path
Specifies the folder for which to get the children. You can specify the as a string or an array of path components. If the path is specified as a string, you can separate path components either with a forward slash or with a semi-colon.
file
Instead of specifying a path you can also specify a file object of the type "folder". When file is specified it will overwrite (and thus, also override) the path property.
showHidden
Boolean to specify whether hidden files should be included in the response
filter
String to specify a search string to filter the result.
The response object passed back to the success callback has one property repositoryFileDto. This is an array representing the list of children. Array items have the following properties:
aclNode
String "true" or "false" to flag if this is an ACL node or not.
createdDate
A string that can be parsed as an integer to get the timestamp indicating the date/time this node was created.
fileSize
A string that can be parsed as an integer to get the size (in bytes) of this node in case this node represents a file. If a filesize is not applicable for this node, it is "-1".
folder
String "true" or "false" to flag if this is node represents a folder or not.
hidden
String "true" or "false" to flag if this is node is hidden for the end user or not.
id
A GUID identifiying this node.
locale
The current locale used for localized properties like title.
localeMapEntries
This is an array of localized properties for this file. The array items have these properties:
locale
The name of the locale for this map of localized properties. There is also a special "default" locale indicating the current locale.
properties
This is a bag of name/value pairs representing localized properties.
key
The key for this property.
value
The localized value for this property.
locked
String "true" or "false" to flag if this is node is locked or not.
name
The name of this node.
ownerType
A string that can be parsed as an integer indicating the owner type.
path
A string containing the forward slash separated path components.
title
The title for presenting this node to the user.
versioned
String "true" or "false" to flag if this is node is versioned or not.
versionId
If the file is versioned, the versionId property is present and its value is a String that represents the version number.

Parameters:

  • conf Object

    Object specifies where to get children from

Defined in src/Phile.js:491

This method can be used to get the entire tree of files. The configuration object supports all generic configuration properties as documented in the request() method. Specific properties:

path
Specifies the folder for which to get the children. You can specify the as a string or an array of path components. If the path is specified as a string, you can separate path components either with a forward slash or with a semi-colon.
file
Instead of specifying a path you can also specify a file object of the type "folder". When file is specified it will overwrite (and thus, also override) the path property.
showHidden
Boolean to specify whether hidden files should be included in the response
filter
String to specify a search string to filter the result.
depth
Integer. Specifies the number of levels to traverse. Default: 1.
The response object passed back to the success callback has two properties:
file
The file property is an object type which holds the properties of the file that is the root of the tree, i.e. the properties of the file identified by the path specified in the argument conf object.
children
Optional. The children property is an array of objects representing the children of the file identified by the path specified in the argument conf object. Each of these objects again has a file property, and optionally, a children property.
.

request

(
  • conf
)
DOMDocument

Defined in src/Phile.js:330

This is the core method to do a request to the Pentaho file service. Normally, you do not need to use this method directy, rather use a specific method like getChildren(), getTree(), getContents(), getProperties(), or save().

The method takes one conf argument which represents the message to send to the service. The conf argument supports the following generic properties:

success
A callback function to be called when the service responds successfully. The callback is passed the following arguments:
options
The conf object that was passed to the request method, i.e., the request.
xhr
The XMLHttpRequest used for this request. Useful if you want to get response headers or process the raw response.
response
An object that represents the response. Its structure is (obviously) dependent upon the specific request.
failure
A callback function to be called when the service did not respond successfully. The callback is passed the following arguments:
options
The conf object that was passed to the request method, i.e., the request.
xhr
The XMLHttpRequest used for this request. Useful if you want to get response headers or process the raw response.
exception
An object that represents the error
scope
The scope to use for the callbacks (defaults to null).
headers
An object to hold any HTTP request headers. The property names of this object are used as HTTP header names, and property values are used as the corresponding header values.
params
An object to hold any HTTP query parameters. The property names of this object are used as parameter names, and their corresponding values as parameter values.
data
The message body to send to the server.

Parameters:

  • conf Object

    An object representing the message.

defaultOptions

Object static

Defined in src/Phile.js:176

These are the default options used for new Phile instances. These can always be overriden by passing an options object to the constructor It sets the following properties:

  • requestTimeout int: 30000 - number of milliseconds before a request to the Pentaho repository service will timeout
  • async boolean: true - determines whether synchronous or asynchronous communication will be used.
  • webapp String: "pentaho" - Name of the pentaho web application. You might need to change this if you changed the default name of the pentaho web application.
  • mountPoint String: "api" - Root of all api webservices. Probably won't need to change this ever.
  • dirService String: "repo/dirs" - Entrypoint for the dirs webservices. See http://javadoc.pentaho.com/bi-platform500/webservice500/resource_DirectoryResource.html
  • fileService String: "repo/files" - Entrypoint for the files webservices. See: http://javadoc.pentaho.com/bi-platform500/webservice500/resource_FileResource.html
  • sessionService String: "session" - Entrypoint for the session webservices. See: http://javadoc.pentaho.com/bi-platform500/webservice500/resource_SessionResource.html
